Git - Github

Publisert av Thomas Nilsen, 02.10.2018

Effektiv bruk av versjon kontroll er viktig og nyttig for utviklere som jobber med kort- og langstidsprosjekter, spesielt hvis det er flere utviklere på samme prosjekt. Det gir et oversiktelig bilde over hvilke endringer som har blitt gjort på et prosjekt. Vi skal nå se nærmere på hva Git og Github er.

Git

Git er et distribuert system for versjonskontroll og kodedeling. Det benyttes av utviklere for å dele prosjekter med full historikk over endringer som har blitt gjort på et spesifikt prosjekt. Ved distribuert menes det at hver enkelt bruker har en lokal kopi av kodebasen på det felles repositoryet, og koden sendes mellom forskjellige brukere. Git har ikke fokusert på et grafisk grensesnitt, men har et godt rykte på seg for å være et raskt og effektivt program som kan håndtere store mengde data uten å bli tregere. Siden Git lagres lokalt, så er det ikke nødvendig å være koblet til et nettverk. Alt kan gjøres offline, frem til det skal deles med andre.

Versjonskontroll er nyttig for å holde oversikt over hvilken forandringer som har blitt gjort i en fil, eller et sett med filer, slik at man lett kan hente tilbake tidligere versjoner ved eventuelle feil. Det gjør det også mulig og legge inn kommentarer undeveis i prosjektet, og man kan benytte forskjellige utviklingsgrener slik at man ikke kræsjer i andres arbeid.

Github

Github kan sammenlignes med Facebook, bare for utviklere. Det er et fullverdig system som gjør at utviklere kan dele et prosjekt med andre medlemmer, slik at de kan få tilbakemeldinger og tips til endringer. Det er en typisk «open source» plattform, samme som Git, bare på nett.

Github er en fullverdig Git-tjeneste bare at den har et mye bedre visuelt grensesnitt og er skybasert. Man behøver ikke en Git klient for å se historikk, gjøre endringer eller kommentere koden i et prosjekt via nettleseren. Systemet fungerer slik at hvis en ønsker å gjøre endring på et prosjekt, så sender man en «pull request». Dette gjør at «host» eller andre som samarbeider i prosjektet, kan se over koden, gjøre endringer og eventuelt implementere dette i prosjektet.
