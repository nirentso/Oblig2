CSS layouts

Publisert av Thomas Nilsen, 29.09.2018

CSS står for Cascading Style Sheets og beskriver hvordan et HTML dokument skal se ut. CSS har egentlig aldri gjort en god jobb på dette, frem til introduksjonen av CSS Grid og flexbox. Et HTML dokument uten noe form for CSS vil vise alt i vertikalt eller horisontal flyt, avhengig av hvordan du skriver koden. Det er her CSS layouts kommer inn, og kan endre posisjonering og hvordan elementer skal flyte. Vi skal nå se nærmere på hva de slags rolle og fordeler de respektive har og hvilken andre typer layout funksjoner som finnes i CSS.

CSS Flexbox

Flexbox er en ny layout metode som ble introdusert i CCS3 som gjør at alt vises «1-dimensjonalt». Metoden benyttes hvis man ønsker at innholdet skal legges i kolonner, eller som rader. Flexbox er ment som en ersatter til funksjoner som "float" og "table" layouts. Som vist under, vil innholdet uten flexbox ta den plassen den krever, men med flexbox vil innholdet tilpasse seg boksen den har fått angitt.

For å enkelt forklare hvordan Flexbox kan bygges opp, er bilder den beste illustrasjonen. Flexbox layout inneholder en «flex container» som igjen har «flex items» som vises under:

«Flex container» kan enten bli strukket horsiontalt eller vertikalt på hovedaksen og «flex items» kan minimeres, bli strukket ut eller fylle plass som ikke er brukt over kryssaksen.

Ved å benytte flexbox kan man lage en ryddig og komplisert layout med kun noen få CSS linjer. For eksempel så vil koden "display:flex" gjøre dette:

  1. Opprette en flex-container
  2. Oppretter flex-items som vil bli lagt fra venstre til høyre som standard
  3. Hvis det ikke er plass til alle «flex items» som ønskes, så vil de automatisk minske i størrelse for å få plass til de horisontalt på en linje.

Det finnes flere muligheter for å endre ved å blant annet benytte flex-wrap, flex-direction, align-content og flex grow.

Alt dette gjør flexbox til et nyttig verktøy for å strukturere opp nettsiden, uten å måtte manuelt sette inn «tables» og benytte «float» for design. I tillegg er det støttet av de fleste browsere, utenom de eldre versjonene av Internet Explorer.

CSS Grid

I CSS flexbox kan man kun behandle kolonner og rader hver for seg. Men med CSS Grid som er et 2-dimensjonalt system, behandles begge samtidig. Dette gjør CSS Grid til et sterkt layout verktøy.

Css Grid behandler nettsiden som et rutenett som gjør at man kan legge til kolonner og rader, som gjør at man lett kan bygge opp layout uten å benytte «juks» med div og span for den samme jobben.

CSS Grid er bygd opp av flere elementer, blant annet:

  1. Grid container - Oppretter rutenett
  2. Grid items - Samme som flexbox
  3. Grid gap - Distanse mellom kolonner og rader
  4. Grid cell - Distanse mellom 2 tilstøtende rader
  5. Grid Area - Celler som utgjør et rektangulært område

Hovedforskjellen mellom Css grid og flexbox er at førstenevnte så kan man behandle kolonner og rader samtidig, som et 2-dimensjonalt system, men i sistenevnte kan man kun behandle dette hver for seg. Men, hvis man kombinerer flexbox og grid, vil man ha et sterkt verktøy for å bygge en layout. Man kan for eksempel benytte Css grid for body layout, så strukturere opp «aside» eller «main» med flexbox.
